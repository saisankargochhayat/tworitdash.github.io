Our Atom Bot

Jun 28th, 2014 | Comments

The name is inspired from Real Steel. This is just a prototype made using ruby libraries, arduino microcontroller board and raspberry pi. The idea is to control a system or hardware from anywhere around the globe by the help of internet. What are actually the ruby libraries and how it helps?

The answer lies within the library files. The gems that are used to make the server-client communication are eventmachine and em-websocket(“eventmachine-websocket”). Websocket, as the name signifies, is a power source, which is web-based. The server side code for the app (web-page) is written in ruby with the help of eventmachine websocket. When the app runs in a terminal, the client or the web page can be connected to it by opening that port in any browser. Or if that is already deployed in any website which supports em-websocket, the port can be conneted through a client code written in ruby with the help of em-websocket. The client can be connected even while using the localhost. How does browser know which port is for the websocket and how to connect it or how does the client code know the same? In the html code for that web-page a script is written which opens the websocket at the same port as written in the server side ruby app code. And in the client ruby code it is also mentioned where to open that websocket port.

PING PONG

The ping-pong is the concept of receiving and sending messages between server and client. The message can be recieved or sent over the websocket. So, we can connect various buttons on that static html page and send commands over the web-socket when the switches are pressed(As we do in normal electric switches over electrical network). And those commands or messages can be received from the client ruby code. And we can also do the reverse. Why to send commands over buttons and receive through a client code?

SerialPort

Here comes another ruby gem called the serialport. This gem is built to use RS-232 serialports. With the help of this gem we can send commands serially through a serialport(where the hardware or the arduino is connected). We can merge the code for serialport within that same client code for the websocket server. In this way we can send that very command (which is being sent from that web-page or browser via web-socket) to the serialport. But why to send commands to the serialport?

Arduino

Arduino is a microcontroller board which has UART (universal asynchronous receiver and transmitter). This UART helps that board to receive the commands coming to it over the serialport. After getting the command(the command I am referring here is a character constant like ‘a’), it can process the output accordingly. How can it know what to perform while that exact command is received? It can all be written in the Arduino IDE and uploaded to that board to do the required tasks. Isn’t that awesome? Arduino has IO ports and with the help of those anything at the output can be connected.

RaspberryPi

RaspberryPi is a credit-card sized computer (linux based) which can be easily connected to the arduino through its serialport(USB) and it can be connected to the internet over wifi. This can be the device in which the client code can be executed. :) :)

A Prototype

We have made a prototype using the above libraries and the set of hardwares. We have connected the raspberrypi to internet through ssh (as we don’t have a separate monitor for that) and connected the arduino with the pi through its serialport. And to the output of arduino, we have connected two motors (the motors are connected to two wheels) with a motor driver circuit. The app now can be executed in any laptop and hence the port can be active so that the websocket port can be opened up by using any browser connected to that internet network or wifi. Then by the above process we can just hit the buttons and see the robot-car moving. And an additional feature that we have included in the script of that html code is the accelerometer. That means any device having the accelerometer feature can run that robot like a game (just moving it sideways). :) Hence the name of our robot-car. And if the websocket server is running on the web, we can control it from anywhere around the globe. The video of our robot-car:
